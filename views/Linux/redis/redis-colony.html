    # 安装过程：
    1.安装ruby ，创建集群需要用到
yum -y install ruby ruby-devel rubygems rpm-build
gem install redis 

Centos 6.5在使用yum -y install ruby 安装ruby的时候，版本号会低于2.2.2. ，在搭建redis集群环境时，执行gem install redis命令时提示以下错误信息：
ERROR:  Error installing redis:
	redis requires Ruby version >= 2.2.2.
ruby --version  // 结果ruby 1.8.0.0p648 (2013-12-16) [x86_64-linux]

解决方法
下载ruby安装包进行升级。
上传tar.gz包到服务器
tar -zxvf ruby-2.3.1.tar.gz
./configure -prefix=/usr/local/ruby
make && make install
cd /usr/local/ruby
cp bin/ruby /usr/local/bin
cp bin/gem /usr/local/bin

    #首先在一台机器上创建 Redis 节点，创建名为7000、7001、7002的目录，并将 redis.conf 拷贝到这三个目录中，并修改redis.conf文件

cd redis_cluster
mkdir 7000 7001 7002
cp /root/redis-3.2.4/redis.conf redis_cluster/7000
cp /root/redis-3.2.4/redis.conf redis_cluster/7001
cp /root/redis-3.2.4/redis.conf redis_cluster/7002

vim redis.conf
port  7000              #端口7000,7001,7002        
bind 本机ip         #默认ip为127.0.0.1 需要改为其他节点机器可访问的ip 否则创建集群时无法访问对应的

daemonize    yes        #redis后台运行

appendonly  yes        #aof日志开启  有需要就开启，它会每次写操作都记录一条日志

    # 无法加入集群故障：

    有一次网卡IP地址变了，导致重启无法加入集群，是因为redis配置bind需要一个IP地址，所以地址变化导致无法组成集群，这就需要重新组建集群了。

    简单说一下原理 redis cluster在设计的时候，就考虑到了去中心化，去中间件，也就是说，集群中的每个节点都是平等的关系，都是对等的，每个节点都保存各自的数据和整个集群的状态。每个节点都和其他所有节点连接，而且这些连接保持活跃，这样就保证了我们只需要连接集群中的任意一个节点，就可以获取到其他节点的数据。 Redis 集群没有并使用传统的一致性哈希来分配数据，而是采用另外一种叫做哈希槽 (hash slot)的方式来分配的。redis cluster 默认分配了 16384 个slot，当我们set一个key 时，会用CRC16算法来取模得到所属的slot，然后将这个key 分到哈希槽区间的节点上，具体算法就是：CRC16(key) % 16384。所以我们在测试的时候看到set 和 get 的时候，直接跳转到了7000端口的节点。 Redis 集群会把数据存在一个 master 节点，然后在这个 master 和其对应的salve 之间进行数据同步。当读取数据时，也根据一致性哈希算法到对应的 master 节点获取数据。只有当一个master 挂掉之后，才会启动一个对应的 salve 节点，充当 master 。

    需要注意的是：必须要3个或以上的主节点，否则在创建集群时会失败，并且当存活的主节点数小于总节点数的一半时，整个集群就无法提供服务了。

    查考资料：https://www.zybuluo.com/phper/note/195558

    单机模式下需搭建的是伪分布式集群，即一台服务器虚拟运行6个redis实例，修改端口号为（7001-7006）；因为Redis集群至少需要3个节点，而主从都在一台机所以需要单机6个节点。 参考资料：https://blog.csdn.net/qq_42815754/article/details/82912130
